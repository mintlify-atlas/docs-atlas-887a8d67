---
title: "Language configuration and plugin settings"
description: "Each of Transmute's six supported languages has a YAML plugin file that defines file extensions, features, frameworks, and syntax mappings."
---

Transmute ships with six language plugin files under `config/languages/`. Each file teaches the translation pipeline what file types belong to that language, which language features to detect, how to map constructs across languages, and metadata about the language's type system and paradigm.

## Supported languages

The six language plugins are:

| File | Language | Version | Extensions |
|---|---|---|---|
| `python.yaml` | Python | 3.11 | `.py`, `.pyw`, `.pyi` |
| `java.yaml` | Java | 17 | `.java` |
| `javascript.yaml` | JavaScript | ES2023 | `.js`, `.jsx`, `.mjs`, `.cjs` |
| `typescript.yaml` | TypeScript | 5.3 | `.ts`, `.tsx` |
| `csharp.yaml` | C# | 12.0 | `.cs` |
| `php.yaml` | PHP | 8.3 | `.php` |

## YAML structure

Every language config file follows the same structure:

```yaml
name: <language_id>         # Internal identifier (lowercase)
display_name: <Human Name>  # Display label shown in CLI output
version: "<version>"        # Target version string

extensions:                 # File extensions this language owns
  - .ext

features:                   # Language features the Analysis Agent detects
  - feature_name

keywords:                   # Reserved words (used for lexical analysis)
  - keyword

frameworks:                 # Commonly detected frameworks and libraries
  - framework_name

build_tools:                # Build system identifiers
  - tool_name

package_managers:           # Package manager identifiers
  - manager_name

syntax_mappings:            # Cross-language construct equivalences
  source_construct: "target_construct"

metadata:                   # Language-level characteristics
  typing_system: "static|gradual|dynamic"
  paradigm: "object-oriented|multi-paradigm|..."
  memory_management: "automatic|manual"
  compiled: true|false
```

## Syntax mappings

The `syntax_mappings` section documents how constructs in one language correspond to constructs in another. These mappings help the Translation Agent choose idiomatic equivalents rather than literal translations.

For example, Python's `list_comprehension` maps to `"for loop with append"` in languages like Java that lack comprehension syntax. Similarly, Python's `with_statement` maps to `"try-with-resources"` in Java.

```yaml
# Python syntax_mappings
syntax_mappings:
  list_comprehension: "for loop with append"
  generator: "iterator"
  decorator: "annotation/wrapper"
  with_statement: "try-with-resources"

# Java syntax_mappings
syntax_mappings:
  interface: "protocol/abstract_class"
  annotation: "decorator"
  generic: "type_parameter"
  stream: "generator/comprehension"

# C# syntax_mappings
syntax_mappings:
  property: "getter_setter/attribute"
  delegate: "function_pointer/callback"
  linq: "query_expression"
  nullable: "optional_type"
  record: "immutable_class/data_class"
```

## Metadata fields

The `metadata` block describes language-level characteristics that the Planning Agent uses when selecting a translation strategy.

| Field | Values | Effect |
|---|---|---|
| `typing_system` | `static`, `gradual`, `dynamic` | Languages with different typing systems (e.g., Python → Java) trigger the Refactor strategy for type-heavy code. |
| `paradigm` | `object-oriented`, `multi-paradigm` | Paradigm shifts increase complexity scores and may require structural reorganization. |
| `memory_management` | `automatic`, `manual` | Affects how resource management patterns are translated (e.g., context managers vs. try-finally). |
| `compiled` | `true`, `false` | Informs build artifact expectations in the output. |

<Note>
  Not all language YAML files include a `metadata` block — JavaScript, TypeScript, C#, and PHP omit it. The pipeline treats missing metadata as unknown and relies on feature detection alone for those languages.
</Note>

## Adding a new language

To add support for a new language, create a YAML file following the structure above in `config/languages/` and register the plugin in the language registry. See [Language plugins](/architecture/language-plugins) for the full plugin architecture.

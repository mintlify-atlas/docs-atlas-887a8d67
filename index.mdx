Transmute is a production-ready CLI tool that migrates entire code repositories between programming languages while preserving functionality, structure, and idioms. Point it at a GitHub repo or local directory, choose a target language, and Transmute's AI pipeline — powered by Claude Sonnet, LangGraph agents, and a RAG system backed by Qdrant — handles the rest.

## How Transmute works

Transmute breaks code migration into four automated stages driven by a LangGraph workflow:

<Steps>
  <Step title="Analyze">
    The Analysis Agent clones your repository, detects the source language, and builds a feature profile using AST parsing with Tree-sitter — identifying classes, decorators, design patterns, frameworks, and a complexity score.
  </Step>
  <Step title="Plan">
    The Planning Agent selects the optimal translation strategy based on code complexity and feature mappings, then creates a per-file migration plan with individual translation units.
  </Step>
  <Step title="Translate">
    The Translation Agent calls Claude Sonnet for each file, optionally augmenting prompts with similar patterns retrieved from the Qdrant RAG system for context-aware, idiom-preserving output.
  </Step>
  <Step title="Validate">
    The Validation Agent scores each translated file for syntax correctness, structural completeness, and translation quality, producing a confidence metric for every unit.
  </Step>
</Steps>

Transmute is a production-ready CLI tool that translates code repositories between programming languages while preserving functionality, structure, and idioms. It combines Claude Sonnet, LangGraph orchestration, and a retrieval-augmented generation (RAG) system to handle everything from simple scripts to complex codebases with paradigm shifts. This page covers what Transmute does, the languages it supports, its key capabilities, and the technology stack that powers it.

## What Transmute does

Transmute clones a repository, analyzes its code with AST parsing and complexity metrics, selects the best translation strategy, and outputs translated files to a directory of your choice. Rather than a simple find-and-replace approach, Transmute uses specialized AI agents for each phase—analysis, planning, translation, and validation—coordinated by a LangGraph state machine.

The result is translated code that respects target-language idioms: type annotations when moving from Python to Java, proper module patterns when targeting JavaScript, and so on.

## Key capabilities

<CardGroup cols={2}>
  <Card title="Multi-agent system" icon="users">
    Four specialized agents handle analysis, planning, translation, and validation independently, each focused on its phase of the pipeline.
  </Card>
  <Card title="RAG-powered translation" icon="database">
    A Qdrant vector database stores translation patterns. The Pattern strategy retrieves similar examples at runtime to improve output quality.
  </Card>
  <Card title="Four translation strategies" icon="shuffle">
    Direct, Pattern, Refactor, and Hybrid strategies are selected automatically based on code complexity, or you can specify one explicitly with `--strategy`.
  </Card>
  <Card title="AST analysis" icon="tree">
    Tree-sitter parses source code into abstract syntax trees for deep structural understanding, symbol table generation, and dependency graph mapping.
  </Card>
  <Card title="Human-in-the-loop (HITL)" icon="user-check">
    Enable manual review checkpoints with `--hitl` to inspect and approve translations before they are written to disk.
  </Card>
  <Card title="Quality validation" icon="check-circle">
    The Validation agent scores each translated file with a confidence metric. Files below the threshold are flagged or retried automatically.
  </Card>
</CardGroup>

## Technology stack

| Component | Technology | Role |
|-----------|-----------|------|
| AI model | Claude Sonnet 4 (`claude-sonnet-4-20250514`) | Code translation and reasoning |
| Orchestration | LangGraph 0.2.55+ | State-driven agent workflow with checkpointing |
| Vector database | Qdrant 1.12+ | RAG pattern storage and similarity search |
| AST parsing | Tree-sitter 0.23+ | Source code structural analysis |
| Embeddings | sentence-transformers 3.3+ | Vector embedding for RAG |
| CLI framework | Typer + Rich | Command-line interface and progress display |

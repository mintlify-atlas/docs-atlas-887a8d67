---
title: "Quickstart: translate your first repository with Transmute"
description: "Install Transmute with Poetry, configure your Anthropic API key, start Qdrant with Docker, and run your first repository translation in five minutes."
---

This guide walks you through everything you need to run your first translation: installing prerequisites, configuring your API key, starting Qdrant, and translating a real open-source Python repository to Java. By the end you will have translated files in your output directory.

<Steps>
  <Step title="Install prerequisites">
    Transmute requires Python 3.12 or later, Docker (for the Qdrant vector database), and Poetry for dependency management.

    Verify each is available:

    ```bash
    python --version     # must be 3.12+
    docker --version
    poetry --version
    ```

    If Poetry is not installed, follow the [official Poetry installation guide](https://python-poetry.org/docs/#installation).

    <Info>
      Docker is only required if you want RAG-powered translation (recommended). You can skip it and use the `--no-rag` flag instead. See the note at the bottom of this page.
    </Info>
  </Step>

  <Step title="Clone and install Transmute">
    Clone the repository and install its dependencies with Poetry:

    ```bash
    git clone https://github.com/Oussiddi/Transmute.git && cd Transmute && poetry install
    ```

    Poetry creates a virtual environment and installs all dependencies defined in `pyproject.toml`, including LangChain, LangGraph, Qdrant client, Tree-sitter, and the Anthropic SDK.
  </Step>

  <Step title="Set your Anthropic API key">
    Transmute uses Claude Sonnet 4 for translation. Export your Anthropic API key as an environment variable:

    ```bash
    export ANTHROPIC_API_KEY="your_api_key_here"
    ```

    To make this persistent across sessions, add the export line to your shell profile (e.g., `~/.bashrc` or `~/.zshrc`).

    <Warning>
      Translation will not start without a valid `ANTHROPIC_API_KEY`. You can get a key from the [Anthropic console](https://console.anthropic.com/).
    </Warning>
  </Step>

  <Step title="Start Qdrant">
    Qdrant is the vector database that powers pattern-based translation. Start it with Docker Compose from the project root:

    ```bash
    docker-compose up -d
    ```

    This pulls the `qdrant/qdrant:latest` image and starts the service on port `6333`. The container restarts automatically unless you stop it manually.

    Verify it is running:

    ```bash
    curl http://localhost:6333/
    ```
  </Step>

  <Step title="Run your first translation">
    Translate the [python-patterns](https://github.com/faif/python-patterns) repository to Java, limiting to 5 files so the first run completes quickly:

    ```bash
    transmute translate https://github.com/faif/python-patterns --to java --max-files 5
    ```

    Transmute analyzes the repository, selects a strategy, translates each file, and validates the output. You will see progress similar to this:

    ```
    ✓ Analysis complete: Python repository (complexity: 0.43)
    ✓ Created plan: 5 units, strategy=pattern
    ✓ Translation complete: 5/5 files (100% success rate)
    ✓ Files saved to: output/
    ```
  </Step>

  <Step title="View your results">
    Translated files are written to the `output/` directory by default. Use `--output` to specify a different location:

    ```bash
    ls output/
    ```

    The summary at the end of each run shows the number of files translated, the success rate, and a confidence score per file. Open any translated file to inspect the Java output.
  </Step>
</Steps>

<Note>
  **No Qdrant? Use `--no-rag`**

  If you cannot run Docker, skip the Qdrant step and pass `--no-rag` to disable pattern retrieval. Transmute falls back to the Direct or Refactor strategy without vector search:

  ```bash
  transmute translate https://github.com/faif/python-patterns --to java --max-files 5 --no-rag
  ```

  Translation quality may be lower for complex codebases without RAG. See [Translation strategies](/guides/translation-strategies) for details on when each strategy is used.
</Note>
